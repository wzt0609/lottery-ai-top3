# GitHub 云端部署说明

这个方案会让 GitHub Actions 在云端运行预测程序，并把结果发布到 GitHub Pages。电脑可以关机，手机用流量也能打开固定链接。

## 效果

部署完成后，你会得到类似这样的链接：

```text
https://你的GitHub用户名.github.io/仓库名/
```

手机打开这个链接即可查看最新预测。

## 自动运行时间

工作流文件：`.github/workflows/lottery-cloud.yml`

- 北京时间 20:05：生成预测页面
- 北京时间 22:05：抓取开奖结果，复盘优化，再发布页面
- 也可以在 GitHub 的 Actions 页面手动点击 `Run workflow`

## 第一次部署步骤

1. 登录 GitHub。
2. 新建一个仓库，例如 `lottery-predictor`。
3. 把本文件夹里的代码上传到这个仓库。
4. 进入仓库的 `Settings` -> `Pages`。
5. Source 选择 `Deploy from a branch`。
6. Branch 选择 `main`，Folder 选择 `/docs`。
7. 保存。
8. 进入仓库的 `Actions` 页面，打开 `Lottery Cloud Update`，点击 `Run workflow`。

运行完成后，GitHub Pages 链接会显示最新页面。

## 能分析哪些信息

当前程序会尝试使用：

- 17500 历史开奖页
- 中国福彩网/中国体彩网公开开奖源
- 17500 首页可抓到的开机号、试机号、关注码
- 走势图统计特征：和值、跨度、奇偶、大小、012 路、重号、和值尾
- 马尔科夫链：按位一阶/二阶转移
- 贝叶斯后验：按位置数字概率平滑
- 形态状态转移：和值区间、跨度区间、奇偶大小状态

专家预测、胆码图、字谜这类内容不是官方开奖信息，且网站结构经常变化。后续可以把稳定可访问的专家页加入 `expert_sources`，但必须以“可回测权重”方式接入，避免某个专家文本把模型带偏。

## 重要限制

彩票开奖结果具有随机性。云端模型只能做统计分析、抓取公开信息、复盘优化，不保证中奖。
